#### 人机识别

- ★★☆☆☆ 哪些场景容易出现机器自动化？

提示：功能自动化（自动聊天、游戏外挂、恶意点击等）、稀缺抢购类（抢红包、秒杀等）、数据爬取类等

- ★★☆☆☆ 端上有哪些常用的自动化实现？

提示：按键精灵、触动精灵、Auto.js（iOS）、UI Automation（iOS）、伪造指令（辅助功能/sendevent/构造点击事件）、无障碍模式、adb、Instrumentation单元测试框架、Hook、API接口调用、物理点击（点击器/筋膜枪）、AI+iOT

- ★★☆☆☆ 抢购类人机识别有哪些策略比较有效？

提示：操作链路是否符合正常业务逻辑（还未开始就调用最终抢购接口等、未经过正常业务入口进入等）；操作累计行为是否符合正常人（1s点击次数、单日观看次数、多日累计交易额等）；精细化对抗（多层CNN-BASE网格手势路径识别、利用加速度陀螺仪传感器数据进行AHRS算法求解姿态角）等。

#### 架构与体系设计

- ★★★★☆ 如何设计一套支撑亿级流量的实时威胁感知架构？

  提示：流式处理（Kafka + Flink）、多级规则引擎（轻规则边缘、重规则中心）、特征向量化（减少计算开销）、背压控制与熔断、分钟级到秒级告警时延优化。

- ★★★★☆ 如何设计 SOAR 平台与现有安全系统的集成策略？

  提示：API 标准化适配层、剧本（Playbook）设计（自动研判→自动处置→人工干预分层）、置信度驱动的自动化程度（高置信自动封禁、中置信告警人工）、回滚机制。

- ★★★★☆ 如何评估一套威胁感知系统的有效性？有哪些量化指标？

  提示：检测率（已知攻击覆盖比例）、误报率（告警中真实威胁占比）、平均检测时间（MTTD）、平均响应时间（MTTR）、未感知攻击占比（通过红队演练验证盲区）。

- ★★★★☆ 云原生环境下，威胁感知的数据采集和分析有哪些挑战？

  提示：容器生命周期短（秒级 Pod 消失导致日志丢失）、服务网格（mTLS 加密流量难以旁路采集）、eBPF 内核级采集（低开销、高覆盖）、云 API 日志（CloudTrail/SLS）整合。

- ★★★★★ 如何建立一套能够检测 APT 组织攻击的威胁感知体系？

  提示：基于 ATT&CK 框架覆盖攻击技战术、行为基线长周期建模（区分正常与异常）、跨维度关联（网络 + 主机 + 身份 + 应用）、情报驱动的主动狩猎（Threat Hunting）、C2 通信模式识别（低频/模拟正常流量）。
